OBJECTIF : Implémenter un système de logs complet dans l’application avec un viewer intégré en temps réel, lisible et performant.

────────────────────────────────────────
1. LOGGER CENTRALISÉ (CORE)
────────────────────────────────────────

Créer un système de logging global remplaçant tous les print/debugPrint existants.

Créer :
- class AppLogger
- class LogEntry

LogEntry doit contenir :
- message (String)
- level (INFO, DEBUG, WARN, ERROR)
- timestamp (DateTime)
- optional: tag (ex: DOWNLOAD, EXTENSION, WEBVIEW)

AppLogger doit :
- exposer un Stream<LogEntry> (broadcast)
- méthode log(message, level, tag)
- méthodes helper :
  - info()
  - debug()
  - warn()
  - error()

Chaque log doit :
- être envoyé dans le stream
- être affiché dans debugPrint

────────────────────────────────────────
2. INTÉGRATION GLOBALE
────────────────────────────────────────

Remplacer tous les logs existants dans l’app :
- download manager
- extensions loader
- webview
- repo fetch
- errors

Par :
AppLogger.log(...) ou AppLogger.error(...)

────────────────────────────────────────
3. LOG VIEWER (UI)
────────────────────────────────────────

Créer un écran : LogScreen

Fonctionnalités :
- affichage en temps réel (stream listener)
- liste scrollable
- auto-scroll activé par défaut
- affichage format :
  [HH:mm:ss] [LEVEL] [TAG] message

Couleurs :
- ERROR → rouge
- WARN → orange
- INFO → vert
- DEBUG → gris

UI :
- AppBar avec :
  - bouton clear logs
  - bouton toggle auto-scroll
  - bouton export

────────────────────────────────────────
4. FILTRAGE & RECHERCHE
────────────────────────────────────────

Ajouter :
- filtre par niveau (ALL / INFO / WARN / ERROR / DEBUG)
- filtre par tag (dropdown ou chips)
- barre de recherche (texte)

Les logs affichés doivent être filtrés en temps réel.

────────────────────────────────────────
5. PERFORMANCE (IMPORTANT)
────────────────────────────────────────

Limiter la mémoire :
- max 1000 logs en mémoire
- supprimer les plus anciens automatiquement

Optimiser :
- éviter rebuild complet de la liste
- utiliser ListView.builder

────────────────────────────────────────
6. ACTIONS UTILISATEUR
────────────────────────────────────────

Sur chaque log :
- long press → copier le log

Global :
- clear logs
- export logs en fichier .txt (local storage)
- share logs (optionnel)

────────────────────────────────────────
7. AMÉLIORATIONS VISUELLES
────────────────────────────────────────

- utiliser une police monospace
- espacement propre
- fond sombre (mode debug)
- possibilité de collaps logs identiques (optionnel)

────────────────────────────────────────
8. INTÉGRATION DANS L’APP
────────────────────────────────────────

Ajouter accès au LogScreen :
- via menu debug
OU
- via Paramètres > Avancé
OU
- gesture caché (optionnel)

────────────────────────────────────────
9. RÉSULTAT ATTENDU
────────────────────────────────────────

- logs visibles en temps réel dans l’app
- lisibilité humaine claire
- outil de debug intégré type Android Studio console
- performant même avec beaucoup de logs
